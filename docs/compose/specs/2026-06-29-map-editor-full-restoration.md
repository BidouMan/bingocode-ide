## [S3] 实施阶段

### Phase 1: 瓦片渲染核心 (P0)
**目标**: 让画布能显示绘制的瓦片图像

1. MapCanvas 添加瓦片渲染层 (PIXI.Container per layer)
2. 从 mapResources 加载图块集图片，按 tileWidth/tileHeight 切片
3. 绘制时渲染瓦片精灵 (从切片中取对应 tile)
4. 支持多图层按顺序渲染
5. 图层可见性控制渲染
6. 瓦片缓存机制 (避免重复加载)

### Phase 2: 资源面板 (P0-P1)
**目标**: 资源列表显示缩略图，支持上传/删除

1. 修正 getTileCount() - 从图像尺寸计算
2. 瓦片缩略图渲染 (切片显示在 tile-cell 中)
3. 资源上传对话框 (图像模式/图块集模式 + 尺寸选择)
4. 资源删除/清空功能
5. 拖拽资源到画布 (可选)

### Phase 3: 工具完善 (P1)
**目标**: 填充工具 + 快捷键

## [S4] 架构决策

### 瓦片渲染架构
- 使用 PixiJS Container 按图层组织瓦片
- 每个图层一个 Container，控制 visible 和 zIndex
- 瓦片精灵使用缓存的 Texture (从图块集切片)
- 绘制/擦除操作直接操作 Container 中的子元素

### 数据流
```
用户绘制 → mapStore.setTile(x,y,tileId) 
  → watch(mapData) → MapCanvas 重新渲染对应瓦片
```

### 资源管理
- mapResources 数组存储当前地图的资源列表
- 每个资源包含: name, path, resourceType, tileWidth, tileHeight
- 上传时复制文件到项目目录，加载为 PIXI.Texture
