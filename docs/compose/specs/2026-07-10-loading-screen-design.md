# Loading页面设计文档

> [!NOTE]
> This document may not reflect the current implementation.
> See the final report for up-to-date state:
> [Final Report](../reports/loading-screen.md)

## [S1] 问题

当前IDE启动时显示纯白页面，用户体验差。需要创建一个美观的Loading页面来提升启动体验。

## [S2] 解决方案

创建一个粒子背景 + Logo动画 + 渐变发光进度条的Loading页面，采用现代简约风格。

## [S3] 技术实现

### 3.1 组件结构

创建独立的Loading页面组件 `LoadingScreen.vue`，包含以下部分：
- Canvas背景：用于渲染粒子效果
- Logo区域：显示BingoCode文字，带呼吸动画
- 进度条区域：显示加载进度

### 3.2 粒子效果实现

使用Canvas API实现简约粒子效果：
- 粒子数量：50-100个
- 粒子样式：白色小圆点，半透明
- 动画效果：缓慢移动，边界反弹
- 性能优化：使用requestAnimationFrame

### 3.3 Logo动画

- 显示BingoCode文字
- 呼吸动画效果：透明度在0.8-1之间循环
- 字体：使用系统字体或项目指定字体

### 3.4 进度条实现

- 渐变色：#5BFB84 → #4B9B5C（项目主题色）
- 发光效果：使用box-shadow实现
- 动画效果：平滑填充

## [S4] 视觉细节

### 4.1 颜色方案

- 背景：深色渐变（#1a1a2e → #16213e）
- 粒子：白色（#ffffff），半透明（opacity: 0.3-0.7）
- Logo：白色，带呼吸动画
- 进度条：渐变色（#5BFB84 → #4B9B5C），带发光效果

### 4.2 布局

- 全屏显示
- Logo居中显示
- 进度条位于底部，宽度60%，居中

### 4.3 动画时长

- 粒子移动：持续动画
- Logo呼吸：3秒循环
- 进度条填充：根据实际加载时间
- 页面过渡：0.5秒淡出

## [S5] 交互细节

### 5.1 加载流程

1. 显示Loading页面
2. 开始加载资源（模拟或实际）
3. 进度条根据加载进度更新
4. 加载完成后进度条填满
5. 整体页面淡出（0.5秒）
6. 显示主界面

### 5.2 热重载支持

为了方便测试，实现以下功能：
- 支持手动触发Loading效果
- 可配置加载时长
- 开发模式下可跳过实际加载

## [S6] 性能考虑

### 6.1 粒子优化

- 粒子数量控制在50-100个
- 使用简单的圆形绘制，避免复杂图形
- 边界检测使用简单数学计算

### 6.2 内存管理

- 组件卸载时停止动画
- 清理Canvas引用
- 避免内存泄漏

### 6.3 渲染优化

- 使用requestAnimationFrame
- 避免频繁的DOM操作
- 合理使用CSS硬件加速

## [S7] 实现步骤

1. 创建LoadingScreen.vue组件
2. 实现Canvas粒子效果
3. 实现Logo呼吸动画
4. 实现渐变发光进度条
5. 集成到App.vue
6. 添加热重载支持
7. 测试和优化

## [S8] 验收标准

1. Loading页面在启动时正确显示
2. 粒子效果流畅运行
3. Logo动画正常工作
4. 进度条正确显示加载进度
5. 加载完成后平滑过渡到主界面
6. 热重载功能正常工作
7. 性能良好，无卡顿
